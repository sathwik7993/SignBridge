# Contributing to SignBridge

Thank you for your interest in contributing to SignBridge! We believe that making the web more accessible for the Deaf and hard-of-hearing community is a collaborative effort. 

## How to Contribute
1. **Fork the Repository**: Start by forking this repository to your respective GitHub account.
2. **Clone Locally**: Clone your fork to your local machine.
3. **Create a Branch**: Create a new branch for your feature or bugfix (`git checkout -b feature/AmazingFeature`).
4. **Make Changes**: Implement your feature or fix.
5. **Commit**: Commit your changes with a descriptive message (`git commit -m 'Add some AmazingFeature'`).
6. **Push**: Push your branch to GitHub (`git push origin feature/AmazingFeature`).
7. **Pull Request**: Open a Pull Request from your fork into the main repository branch.

## Areas for Contribution
- **NLP Improvements**: Refining the ASL gloss generation to better support proper ASL sentence structures (Time-Topic-Comment formats).
- **Scraper Resiliency**: Improving the `mapper.py` fallback systems when an exact matching video is not found on public dictionaries.
- **Frontend UI/UX**: Making the Chrome Extension overlay more responsive, adding dragging capabilities, or improving the styling.
- **Browser Porting**: Porting the extension over to Firefox or Safari.

Please make sure all code adheres to basic standard style guides and is thoroughly commented. If you encounter bugs, please open an Issue first!
